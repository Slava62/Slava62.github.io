Проекты C#

В моем резюме программирование на языке C# указано в разделе «компьютерные навыки». Не смотря на то, что основной специализацией моей трудовой деятельности является технология механической обработки на станках с ЧПУ, программирование на протяжении длительного периода времени использовалось для выполнения работы по контракту (прикладное ПО), а также решения задач автоматизации выполнения однотипных операций (программы DrillSplitter, Get_xls_data). Знания основы ООП, SQL и Visual Basic for Application я приобрел в процессе моего обучения (факультативные занятия) в Рязанском институте (филиале) МГОУ, где и начал выполнять задачи разработки пользовательского ПО, используя среду разработки Visual Basic 6.0. Были разработаны первая версия программы «Строительная лаборатория», для расчета состава асфальтобетонных смесей, позволяющая рассчитать состав компонентов и получить рецепт для заданного типа смеси. Также была разработана программа Rustravel (в 2005 г.) для туристического агентства – электронный каталог санаторно-курортных учреждений с возможностями демонстрации характеристик учреждения, выполнения фильтрации и сортировки по заданным параметрам. Параллельно (до 2010 г.) проводилась работа по обновлению программы «Строительная лаборатория» с целью удовлетворения требований новых заказчиков, изменений форм отчетности и улучшения функционала. Представленные программы были написаны на VB6.0, в качестве хранилища использовались базы данных MS Access (обращение посредством SQL запросов), вывод результатов производился на листы MS Excel (в программе Rustravel результат отображался в браузере). В 2010 г. было принято решение о переводе программы «Строительная лаборатория» на платформу .NET и использовать C# для реализации. В процессе самостоятельного обучения C# (под руководством коллеги по основному направлению трудовой деятельности, который имел навыки написания программ на C#, а также с использованием специализированной литературы и интернет источников) были закреплены знания принципов ООП. В качестве инструмента разработки использовалась IDE Visual Studio 8. В процессе разработки был изучен функционал объектов C#, применена рекурсия для построения дерева составов, использованы возможности для работы с xml файлами (для хранения и передачи данных), парсирования html файла-шаблона (для вывода результатов в браузер) и др. В 2011 г. были разработаны 2 программы прикладного характера DrillSplitter и Get_xls_data на C#. Первая предназначена для парсирования управляющей программы для станка с ЧПУ(текстовый файл) и вставки определенного кода, с целью разбиения программы на блоки; вторая — для автоматизации сбора неструктурированной информации из входящих документов (в виде файлов MS Excel) и передачи ее в другую программу. В первом случае применялись возможности C# по работе с текстовыми файлами, во втором — работа с событиями стороннего приложения (MS Excel). Параллельно (до 2015г.) продолжалась работа по созданию обновленной версии программы «Строительная лаборатория» (GradingE), но ввиду недостаточного количества времени в связи с загрузкой по основному направлению трудовой деятельности, а также, учитывая объем работы для одного программиста по данному проекту, она, к сожалению, не была завершена. В 2015 г. посредством видео уроков я начал изучать язык JAVA с целью создания интернет сервиса GradingE на основе имеющихся наработок на C#, но к реализации так и не приступил по настоящий момент ввиду отсутствия свободного времени. Более детальное описание представленных программ находится в прилагаемом архиве. Ниже один из примеров программ на C# (остальные добавлю позже)

Программа DrillSplitter

В процессе сверления большого количества отверстий на станках с ЧПУ возникает задача останова выполнения управляющей программы станка и выхода в определенную позицию для контроля состояния сверла. Современные CAM-системы не всегда позволяют вставить блоки кода для разбиения управляющей программы (УП) станка, поэтому их приходится вставлять вручную. В том случае, когда УП рассчитана на 1000 — 2000 отверстий, и выход нужно осуществлять каждые 50 отверстий, а также таких УП много, возникает задача автоматизации вставки блоков. Этот процесс реализован в данной программе.

В качестве исходных данных приложение использует текстовый файл УП. По заданному алгоритму происходит поиск мест вставки блоков. После этого УП можно сохранить. Также есть возможность изменить параметры блока.
